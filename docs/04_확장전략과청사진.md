# 04 · 확장 전략과 청사진 (v0.11)

> 이 문서는 "부품을 어떻게 늘릴 것인가"의 전략과, v0.11에서 실제로 추가한 것·남은 것을 기록한다.
> 세부 부품 규격은 README 5절·`docs/02`. 청사진 대조표(widget-blueprint-registry)는 현재 미동봉(별도 작업).

---

## 1. 전략 — 청사진 수확 → 닫힌 DSL로 조립

부품을 발명하지 않는다. 대신 **검증된 디자인시스템(IBM Carbon · MS Fluent · Shopify Polaris · Ant Design · Google/MUI Material · Atlassian · Mantine · Radix/shadcn)에서 위젯의 *청사진*(해부·표준 props·동작)을 수확**하고, 그것을 우리 닫힌 DSL(토큰·원자·분자·유기체·템플릿)로 *조립*한다.

- 청사진 = vendor-중립 합의형. 여러 시스템에 공통이면 "정형화됨" 신호.
- 적용 사례(예: 색상 팔레트 선택)는 청사진이 아니라 **데이터/리프**로 들어온다 — 컴포넌트가 도메인을 모른다.
- 전체 청사진 ↔ DSL 조립 대조표(약 80종, ERP 중요도 S/A/B)는 현재 미동봉(별도 파일, 추후 복원).

## 2. json-render / A2UI 활용 여지 (엔진은 흡수, 고유층은 유지)

"LLM → 닫힌 카탈로그 → 화면" 메커니즘은 이미 상용 엔진이 있다:
- **Vercel json-render** — Catalog + Registry + Spec(JSON) + Zod. 우리 `_catalog`·`_registry`·`FieldSpec(Zod)`과 동형. **엔진(스펙→화면·검증·스트리밍·데이터 바인딩)을 흡수할 후보.**
- **Google A2UI** — 같은 파이프라인의 *프로토콜* 버전(크로스플랫폼·멀티에이전트). 단일 ERP 제품엔 과하나, 외부 에이전트/멀티플랫폼 개방 시의 **미래 출구**.

**경계(우리 고유, 대체 불가):** ① 토큰 헌법(임의 hex/px 금지) ② `FieldSpec`(함수 없는 직렬화·LLM-safe 도메인 진입) ③ ERP 유기체·템플릿. → 엔진은 빌리고, 이 세 층은 직접 키운다. (스키마-폼 패턴 자체는 JSON Forms·RJSF·Formily·Ant ProForm 선행 — 신규 아님. 우리 차별은 닫힘+함수금지+토큰.)

향후 FieldSpec 확장 후보(미구현, 함수-금지 유지): 조건 술어 일반화(`op: eq|in|gt…` + 다중 필드), `visibleWhen`, 중첩/배열 필드, 캐스케이딩 옵션(`optionsFrom`), 검증 `format` enum.

## 4. 정형화 백로그 (게이트 — dev 확인 후 기존 땜빵 제거)

신규 부품을 *추가*만 했고, 기존 땜빵 **제거·교체는 dev에서 비포/애프터 확정 뒤** 별건으로 진행:

1. **ListPage 죽은 필터 버튼** (`onClick=()=>{}`) → `SearchToolbar`로 교체.
2. **DataTable 로딩** `<Center><Spinner/></Center>` → 실제 columns 기반 `Skeleton` 행.
3. **Collapsible 직접 쌓기** → `Accordion`(공유 open-state).
4. ~~**HierarchyExplorer 손빌드 검색** → `Combobox`/`SearchToolbar`.~~ → **0.17.0에서 재작업 완료**: 검색을 좌측 분류 헤더 아래로 이전(Tree `toolbar` 슬롯) + 결과를 우측 목록(분류 경로 컬럼)으로. controlled `searchResults`(소비처 필터) 유지. Combobox/SearchToolbar 치환은 현 구조로 충족되어 불요.
5. **Modal로 옆 패널** → `Drawer`(맥락 유지 흐름).
6. **notify 호스트가 Providers에 묻힘** → `ToastHost`로 단일화.
7. **Tree-입력 보일러플레이트**(수동 onSelect/expand) → `TreeSelect`/`Cascader`.

## 5. 핵심 설계 결정 (이번 라운드)

- **Cascader = 방식 A+압축(C):** 순차 드롭다운(한 칸→다음 칸, 깊이=칸 수, 폭 오버플로 0) + 완료 시 `라벨 › 라벨 [변경]` 압축. 다단 컬럼 팝오버 폐기. 시각적 리프(색·재질)는 컴포넌트 밖.
- **Tree vs TreeSelect 독립:** Tree=파인더·HierarchyExplorer·셸 네비, TreeSelect=노드를 값으로 고르는 입력. 서로 대체 아님.
- **Collapsible vs Accordion:** 단독 토글=Collapsible, 그룹(하나만/동시)=Accordion(`multiple`). 경쟁경로(쌓기) 금지.
- **Modal vs Drawer:** 뒤 화면 보여야 하면 Drawer, 가려도 되면(차단) Modal.
- **Transfer:** 두 카드는 더 큰 쪽 기준 *동적* 동일 높이(`flex stretch + Card fill`, 고정 수치 X), 헤더·본문 동일 패딩(sm)으로 좌측 정렬.
- **before/after 규칙:** 진짜 기존 땜빵을 대체할 때만. 신규·형제는 단독.
